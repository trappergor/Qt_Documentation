Обзор

Инструмент qmake предоставляет проектно-ориентированную систему для управления процессом сборки приложений, библиотек и других компонентов. Этот подход дает вам контроль над исходными файлами и позволяет кратко описать каждый шаг процесса, обычно в рамках одного файла. qmake расширяет информацию в каждом файле проекта до Мэйкфайла, выполняющего необходимые команды для компиляции и линковки.

Описание проекта

Сведения о проекте описываются в проектном файле (.pro). qmake использует информацию из этих файлов для создания Мейкфайлов, содержащих все команды, необходимые для сборки каждого проекта. Файлы проектов обычно содержат список исходных и заголовочных файлов, общую информацию о конфигурации, а также любую конкретную информацию о приложении, например, список дополнительных библиотек для компоновки или список дополнительных включаемых путей (include paths) использования.

Файлы проектов могут содержать множество различных элементов, включая комментарии, объявления переменных, встроенные функции и некоторые простые структуры управления. В большинстве простых проектов необходимо декларировать только исходные и заголовочные файлы, которые используются для сборки проекта, с некоторыми базовыми настройками. Дополнительные сведения о том, как создать простой файл проекта, см. в разделе Начало работы.

Вы можете создавать более сложные проектные файлы для сложных проектов. Обзор файлов проекта см. в разделе Создание проектных файлов. Подробную информацию о переменных и функциях, которые можно использовать в файлах проекта, см. в разделе Ссылка.

Вы можете использовать проектные или библиотечные шаблоны приложений для задания особых параметров конфигурации для точной настройки процесса сборки. Дополнительные сведения см. в разделе Построение общих типов проектов.

Вы можете использовать мастера проектов Qt Creator для создания проектного файла. Вы выбираете шаблон проекта, и Qt Creator создает файл проекта со значениями по умолчанию, которые позволяют вам собрать и запустить проект. Вы можете модифицировать файл проекта в соответствии со своими целями.

Вы также можете использовать qmake для генерирования проектных файлов. Полное описание опций командной строки qmake смотрите в разделе Запуск qmake.

Базовые возможности Qmake по конфигурированию позволяют работать с большинством кросс-платформенных проектов. Тем не менее, может оказаться полезным или даже необходимым использовать некоторые специфичные для платформы переменные. Дополнительную информацию см. в Примечаниях к платформе.

Построение проекта

Для простых проектов достаточно запустить qmake в директории верхнего уровня вашего проекта, чтобы сгенерировать Мейкфайл. Затем вы можете запустить утилиту make для сборки проекта в соответствии с Мейкфайлом.

Для получения дополнительной информации о переменных окружения, которые qmake использует при настройке процесса сборки, см. раздел Настройка qmake.

Руководство по сторонним библиотекам показывает вам, как использовать простые сторонние библиотеки в вашем проекте Qt.
