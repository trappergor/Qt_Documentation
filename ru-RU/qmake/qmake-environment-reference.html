Настройка qmake

Свойства

qmake имеет систему для постоянной конфигурации, которая позволяет вам установить свойство в qmake один раз, и запрашивать его при каждом вызове qmake. Вы можете установить свойство в qmake следующим образом:

qmake -set PROPERTY VALUE

Соответствующее свойство и значение должны быть подставлены вместо PROPERTY и VALUE.

Эту информацию можно получить обратно из qmake следующим образом:

qmake -query PROPERTY qmake -query #queries all current PROPERTY/VALUE pairs

Замечание: qmake -query перечисляет встроенные свойства в дополнение к свойствам, которые вы задаете с помощью qmake -set PROPERTY VALUE.

Эта информация будет сохранена в объекте QSettings (то есть будет храниться в разных местах для разных платформ).

Ниже приведен список built-in свойств:

  • QMAKE_SPEC - сокращенное имя mkspec хоста, которое обрабатывается и хранится в переменной QMAKESPEC во время сборки хоста.
  • QMAKE_VERSION - текущая версия qmake
  • QMAKE_XSPEC - сокращенное имя объекта mkspec, которое преобразуется и сохраняется в переменной QMAKESPEC во время целевой сборки
  • QT_HOST_BINS - расположение исполняемых файлов хоста
  • QT_HOST_DATA - расположение данных для исполняемых файлов, используемых qmake.
  • QT_HOST_PREFIX - префикс по умолчанию для всех путей хоста.
  • QT_INSTALL_ARCHDATA - расположение общих архитектурно-зависимых данных Qt
  • QT_INSTALL_BINS - расположение двоичных файлов Qt (инструментов и приложений)
  • QT_INSTALL_CONFIGURATION - расположение для настроек Qt. Не применимо для Windows
  • QT_INSTALL_DATA - расположение общих архитектурно-независимых данных Qt
  • QT_INSTALL_DOCS - расположение документации
  • QT_INSTALL_EXAMPLES - расположение примеров
  • QT_INSTALL_HEADERS - расположение всех заголовочных файлов
  • QT_INSTALL_IMPORTS - расположение расширений QML 1.x
  • QT_INSTALL_LIBEXECS - расположение исполняемых файлов, необходимых библиотекам во время выполнения программы
  • QT_INSTALL_LIBS - расположение библиотек
  • QT_INSTALL_PLUGINS - расположение плагинов Qt
  • QT_INSTALL_PREFIX - префикс по умолчанию для всех путей
  • QT_INSTALL_QML - расположение расширений QML 2.x
  • QT_INSTALL_TESTS - расположение тестовых примеров Qt
  • QT_INSTALL_TRANSLATIONS - расположение информации о переводе для строк Qt
  • QT_SYSROOT - системный корень, используемый целевой средой сборки
  • QT_VERSION - версия Qt. Мы рекомендуем запрашивать номера версий модулей Qt, используя переменные $$QT.<module>.version

Например, вы можете запросить установку Qt для этой версии qmake с помощью свойства QT_INSTALL_PREFIX:

qmake -query "QT_INSTALL_PREFIX"

Вы можете запросить значения свойств в файле проекта следующим образом:

QMAKE_VERS = $$[QMAKE_VERSION]

QMAKESPEC

qmake требует файл описания платформы и компилятора, который содержит множество значений по умолчанию, используемых для генерации соответствующих Makefiles. Стандартный дистрибутив Qt поставляется со многими из этих файлов, расположенных в подкаталоге mkspecs установки Qt.

Переменная окружения QMAKESPEC может содержать любое из следующих значений:

  • Полный путь к директории, содержащей файл qmake.conf. В этом случае qmake откроет файл qmake.conf из этого каталога. Если файла не существует, qmake завершит работу с ошибкой.
  • Название комбинации платформа-компилятор. В этом случае qmake будет искать в каталоге, указанном подкаталогом mkspecs пути данных, указанного при компиляции Qt (см. QLibraryInfo::DataPath).

Замечание: Путь QMAKESPEC будет автоматически добавлен в системную переменную INCLUDEPATH.

Файл кэша

Файл кэша представляет собой специальный файл qmake, который содержит настройки, не указанные в файле qmake.conf, файлах проектов или в командной строке. Когда qmake запущен, он ищет файл под названием .qmake.cache в родительских каталогах текущей директории, если только вы не укажете -nocache. Если qmake не найдет этот файл, он автоматически проигнорирует этот шаг обработки.

Если qmake найдет файл .qmake.cache, он сначала обработает этот файл, а затем файл проекта.

Расширения файлов

В обычных условиях qmake попытается использовать соответствующие расширения файлов для вашей платформы. Однако иногда необходимо переопределить настройки по умолчанию для каждой платформы и явно указать расширения файлов для использования qmake. Это достигается путем переопределения некоторых встроенных (built-in) переменных. Например, расширение, используемое для moc-файлов, может быть переопределено следующим образом в файле проекта:

QMAKE_EXT_MOC = .mymoc

Следующие переменные могут быть использованы для переопределения распространенных расширений файлов, распознаваемых qmake:

  • QMAKE_EXT_MOC модифицирует расширение, размещенное на включенных moc-файлах.
  • QMAKE_EXT_UI модифицирует расширение, используемое для файлов пользовательского интерфейса Qt Designer (обычно в FORMS).
  • QMAKE_EXT_PRL изменяет расширение, размещенное на файлах зависимостей библиотек.
  • QMAKE_EXT_LEX изменяет суффикс, используемый в Lex-файлах (обычно в LEXSOURCES).
  • QMAKE_EXT_YACC изменяет суффикс, используемый в файлах Yacc (обычно в YACCSOURCES).
  • QMAKE_EXT_OBJ изменяет суффикс, используемый для генерируемых объектных файлов.

Все вышеперечисленное принимает только первое значение, поэтому вы должны задать лишь одно значение, которое будет использоваться во всем файле вашего проекта. Есть две переменные, которые принимают список значений:

  • QMAKE_EXT_CPP заставляет qmake интерпретировать все файлы с этими суффиксами как исходные файлы C++.
  • QMAKE_EXT_H заставляет qmake интерпретировать все файлы с этими суффиксами как C и C++ заголовочные файлы.
